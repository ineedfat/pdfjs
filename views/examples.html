This example explores basic translation and text manipulation.
var doc = new pdfJS.doc('letter', 'portrait'); var p = doc.addPage(); var drawRect = function (x, y, f, s) { p.rect(x, y, 100, 100); p.fillColor(f); p.strokeColor(s); p.paintPath(); }; p.translate(160, 400); drawRect(50, 50, 1, 0.5); drawRect(75, 75, 0, 0.5); p.beginText(); p.fontSize(22); p.print('Pretty Cool Huh?'); p.endText();

Note

Here shows a basic rect operator that draws a rectangular path by specifying the x-coord, y-coord, width, and height. Drawing the path alone will not actually render anything on the page. We must first specify the stroke and fill color and then finally paint the path by using the paintPath operator. Also note that in this example, we are using the grayscale color space by only specifying one color value to fillColor and strokeColor.

var doc = new pdfJS.doc('letter', 'portrait'); var p = doc.addPage(); var drawRect = function (x, y, f, s) { p.pushState(); p.translate(x, y); p.fillColor(f); p.rect(0, 0, 90, 90); p.lineWidth(3); p.strokeColor(s); p.paintPath(); p.popState(); }; p.translate(0, 400); drawRect(100, 100, .6, 0); drawRect(200, 200, .1, .5); drawRect(300, 100, .9, 0); drawRect(400, 200, .9, 0); p.translate(400, 30); p.beginText(); p.fontSize(22); p.print('Push and Pop :}'); p.endText();

Note

PDF (as well as PostScript) allows graphic state to get pushed (save) on to the stack and pop (restore) off the stack to recover to the original configuration.

var doc = new pdfJS.doc('letter', 'portrait'); var p = doc.addPage(); var imgXObj = doc.newImage('./img/sample.jpg', true); p.addImage(imgXObj, 60, 300, 500, 200); p.translate(400, 475); p.beginText(); p.fontSize(15); p.print('Miyajima, Japan'); p.endText();

Note

PdfJS uses HTML5 canvas internally to support different kinds of image format. The newImage function accepts a string url to an image, HTMLImageElement, or HTMLCavasElement as graphics to render.
